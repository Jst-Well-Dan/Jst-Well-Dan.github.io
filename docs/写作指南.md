# Dan 博客写作指南

## 一、新建文章

```bash
hugo new posts/<slug>/index.md
```

`<slug>` 用英文小写加连字符，例如 `my-new-post`。

Hugo 会在 `content/posts/<slug>/` 下生成 `index.md`，并填入 archetype 模板的默认 front matter。

---

## 二、编辑 Front Matter

打开 `index.md`，补全以下字段：

```yaml
---
title: "文章标题"
date: 2026-01-01
draft: true                      # 发布前改为 false，或直接删除
description: "一句话摘要"
categories: ["工具与技术"]       # 三选一：AI与学习 / 金融与投资 / 工具与技术
tags: ["标签1", "标签2"]
cover:
  image: "cover.png"             # 封面图文件名，放在同目录下
---
```

---

## 三、处理封面图

### 规格要求

| 项目 | 规格 |
|------|------|
| 尺寸 | **1200 × 630 px** |
| 比例 | **1.9 : 1** |
| 格式 | PNG 或 JPG |

所有封面必须保持相同比例，否则列表页卡片高度不一致。

### 裁切方法

将封面图放入文章目录，然后运行：

```bash
python scripts/crop-cover.py content/posts/<slug>/cover.png
```

脚本会**居中裁切**后覆盖原文件，直接输出 1200×630。

> 裁切策略：保留图片中央区域。如果主体偏上或偏下被裁掉，用 Figma / PS 手动调整后再跑脚本。

---

## 四、撰写正文

- 图片用相对路径引用：`![](image-name.png)`，文件放在同一目录下
- **不要**用 Obsidian 的 `![[image.png]]` 语法
- 封面图**不需要**在正文第一行重复插入（front matter 的 `cover.image` 已经处理）

---

## 五、本地预览

```bash
hugo server -D
```

`-D` 参数让草稿（`draft: true`）也能渲染。浏览器打开 `http://localhost:1313` 查看效果。

---

## 六、发布

1. 将 `draft: true` 改为 `false`，或直接删除该行
2. 推送到 `main` 分支：

```bash
git add .
git commit -m "post: <文章标题>"
git push origin master:main
```

GitHub Actions 会自动构建并部署到 `https://jst-well-dan.github.io/`，通常 1-2 分钟完成。

---

## 七、完整检查清单

发布前逐项确认：

- [ ] `title` / `date` / `description` 已填写
- [ ] `categories` 选了三个之一
- [ ] 封面图已裁切为 1200×630，路径写入 `cover.image`
- [ ] `draft` 已删除或改为 `false`
- [ ] 本地 `hugo server -D` 预览无报错
